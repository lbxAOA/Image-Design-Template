# 雷达图 (Radar Chart / Spider Chart)

## 📊 图表简介

雷达图（又称**蜘蛛图**、**星形图**）是从一个中心点向外辐射多条轴线，将多个维度的数据绘制在这些轴线上并连接成多边形的图表，适合多维度综合评估。

## 🎯 适用场景

| 场景 | 说明 |
|------|------|
| **多维度评估** | 产品/方案的多指标综合评分 |
| **能力画像** | 个人/团队的多技能雷达图 |
| **竞品分析** | 多个竞品在各维度的对比 |
| **绩效评估** | KPI多指标达成情况 |
| **质量评价** | 产品质量的多维度展示 |

## 📈 图表分类

### 1. 基础雷达图
- 单一对象的多维度展示
- 快速了解各维度的均衡程度

### 2. 多系列雷达图
- 多个对象叠加对比
- 直观发现各对象的优劣势

### 3. 填充雷达图
- 多边形内部填充颜色
- 面积大小直观反映综合得分

### 4. 玫瑰雷达图
- 用扇形面积代替多边形
- 更强调各维度的数值大小

## ⚖️ 与相似图表的对比

| 对比图表 | 雷达图优势 | 雷达图劣势 |
|----------|-----------|-----------|
| **柱状图** | 一图展示多维度，整体感强 | 维度多时变拥挤，精确读数困难 |
| **表格** | 视觉效果好，易发现不均衡 | 精确数值展示不如表格 |
| **平行坐标图** | 维度顺序影响小 | 维度超过7-8个时效果差 |

## 🔄 雷达图使用注意事项

### 维度设计
- **维度数量**：建议5-8个，最多不超过12个
- **维度顺序**：相关维度放在一起，或按重要性排列
- **数值方向**：确保所有维度"越大越好"或统一处理

### 数据预处理
- **归一化**：不同量纲的数据需要归一化到相同范围
- **方向统一**：如果某指标"越小越好"，需要反转处理

## 🤔 如何选择

```
展示目的？
├── 多维度综合评价 → 雷达图
├── 精确数值比较 → 柱状图或表格
├── 大量维度（>10）→ 平行坐标图
└── 趋势变化 → 折线图

对比对象数量？
├── 1个 → 基础雷达图
├── 2-3个 → 多系列雷达图
└── >4个 → 考虑多图或其他图表
```

## 💡 使用建议

1. **维度控制**：5-8个维度最佳，过多会降低可读性
2. **数据归一化**：确保各维度在同一量级（如0-100）
3. **颜色透明度**：多系列时使用透明填充，避免遮挡
4. **添加数值**：在顶点附近标注具体数值
5. **参考线**：添加同心圆网格作为参考
6. **图例清晰**：多系列时确保图例易于识别
7. **避免误导**：注意维度排列顺序可能影响感知

## ⚠️ 常见问题

| 问题 | 解决方案 |
|------|----------|
| 维度量纲不同 | 数据归一化处理 |
| 多边形重叠严重 | 使用透明度或改用其他图表 |
| 难以读取精确值 | 添加数值标签或配合表格 |
| 维度顺序影响感知 | 固定排列规则或多角度展示 |

## 📐 解读技巧

| 图形特征 | 含义 |
|----------|------|
| 面积大 | 综合表现好 |
| 形状均匀 | 各维度发展均衡 |
| 某个尖角突出 | 该维度表现突出 |
| 某处凹陷 | 该维度表现较弱 |
| 多边形重合度高 | 对比对象表现相似 |
