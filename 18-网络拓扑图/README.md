# 网络拓扑图 (Network Topology Diagram)

## 📊 图表简介

网络拓扑图是用**节点和边**来表示实体及其连接关系的图表，常用于展示网络结构、关系网络、组织架构等复杂系统的结构。

## 🎯 适用场景

| 场景 | 说明 |
|------|------|
| **计算机网络** | 服务器、路由器、交换机的连接拓扑 |
| **社交网络** | 人物关系、社群结构 |
| **知识图谱** | 概念和概念间的关系 |
| **组织架构** | 公司部门、汇报关系 |
| **引用网络** | 论文引用、专利引用关系 |
| **生物网络** | 蛋白质互作、代谢通路 |

## 📈 图表分类

### 按拓扑结构分类

| 拓扑类型 | 特点 | 典型应用 |
|----------|------|----------|
| **星型** | 中心节点连接所有其他节点 | 客户端-服务器 |
| **环型** | 节点首尾相连成环 | 令牌环网络 |
| **总线型** | 所有节点共享一条主干线 | 早期以太网 |
| **树型** | 层次结构，有根节点 | 组织架构 |
| **网状** | 节点间多重连接 | 互联网 |
| **混合型** | 多种拓扑组合 | 企业网络 |

### 按布局算法分类
- **力导向布局**：节点像弹簧一样相互作用
- **层次布局**：按层级排列
- **圆形布局**：节点排列在圆周上
- **网格布局**：节点排列在网格上

## ⚖️ 与相似图表的对比

| 对比图表 | 网络图优势 | 网络图劣势 |
|----------|-----------|-----------|
| **和弦图** | 展示完整网络结构 | 关系强度展示不如和弦图 |
| **桑基图** | 适合任意网络拓扑 | 不适合展示流量 |
| **树状图** | 不限于层次结构 | 层次结构展示不如树状图清晰 |

## 🔄 网络图的设计元素

### 节点属性
| 属性 | 含义映射 |
|------|----------|
| 大小 | 重要性/度中心性 |
| 颜色 | 类别/社群归属 |
| 形状 | 节点类型 |
| 标签 | 节点名称 |

### 边属性
| 属性 | 含义映射 |
|------|----------|
| 粗细 | 关系强度/权重 |
| 颜色 | 关系类型 |
| 样式 | 虚线=弱关系，实线=强关系 |
| 箭头 | 关系方向 |

## 🤔 如何选择

```
展示目的？
├── 网络整体结构 → 网络拓扑图
├── 循环/对称关系 → 和弦图
├── 流量分配 → 桑基图
└── 层次结构 → 树状图

网络规模？
├── 小（<30节点）→ 直接展示
├── 中（30-100节点）→ 需要布局优化
└── 大（>100节点）→ 考虑聚合或采样

关注点？
├── 连接模式 → 力导向布局
├── 层级关系 → 层次布局
├── 节点对称性 → 圆形布局
└── 地理位置 → 地理布局
```

## 💡 使用建议

1. **节点数量控制**：
   - <30个：清晰可读
   - 30-100个：需要交互功能
   - >100个：考虑聚合、过滤或采样
2. **布局算法选择**：
   - 通用：力导向布局
   - 有层级：层次布局
   - 有地理属性：地理布局
3. **颜色编码**：
   - 用颜色区分社群/类别
   - 避免使用过多颜色（≤7种）
4. **边的处理**：
   - 边过多时使用边捆绑
   - 弱连接可以过滤
5. **交互功能**：
   - 缩放、平移
   - 节点拖拽
   - 悬停显示详情
6. **标签显示**：
   - 节点少时直接显示
   - 节点多时悬停显示

## 📐 网络度量指标

| 指标 | 含义 | 可视化方式 |
|------|------|-----------|
| 度中心性 | 连接数量 | 节点大小 |
| 介数中心性 | 桥梁作用 | 节点颜色 |
| 接近中心性 | 到其他节点的距离 | 节点位置 |
| 聚类系数 | 局部密集程度 | 节点形状 |
| 社群 | 节点分组 | 节点颜色 |

## ⚠️ 注意事项

1. **头发球问题**：节点过多时图表变成难以辨认的"头发球"
2. **布局稳定性**：力导向布局每次运行结果可能不同
3. **边交叉**：尽量减少边的交叉
4. **标签遮挡**：标签过多会相互遮挡
5. **性能问题**：大规模网络需要考虑渲染性能
